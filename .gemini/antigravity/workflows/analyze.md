# 🔍 分析・デバッグ

プロジェクトの分析、デバッグ支援、セキュリティ監査を管理します。

## 使い方

```
/analyze project  → プロジェクト構造・技術スタック分析
/analyze debug    → エラー分析とデバッグ支援
/analyze security → セキュリティリスクの監査
```

---

## /analyze project

プロジェクト構造・技術スタック・依存関係を分析します。

### 1. プロジェクト構造の確認
// turbo
```bash
ls -la
cat package.json 2>/dev/null || cat requirements.txt 2>/dev/null
```

### 2. 分析項目
- ディレクトリ構造
- 使用技術・フレームワーク
- 依存パッケージ
- 設定ファイル
- テスト構成

### 3. 結果報告
```markdown
## 📊 プロジェクト分析結果

### 技術スタック
- 言語: [言語]
- フレームワーク: [フレームワーク]
- パッケージマネージャ: [npm/yarn/pip等]

### ディレクトリ構造
[主要ディレクトリの説明]

### 依存関係
[主要な依存パッケージ]

### 推奨事項
[改善提案]
```

---

## /analyze debug

エラー分析とデバッグ支援を行います。

### 1. エラー情報の収集
ユーザーからエラーメッセージ・スタックトレースを取得

### 2. 分析観点
- エラーメッセージの解読
- スタックトレースの分析
- 関連コードの特定
- 類似問題の検索

### 3. デバッグ手順
```markdown
## 🐛 デバッグ分析

### エラー概要
[エラーの簡潔な説明]

### 原因の可能性
1. [可能性1]
2. [可能性2]

### 確認すべきファイル
- [ファイル1]
- [ファイル2]

### 修正案
[具体的な修正案]

### 再発防止策
[同様の問題を防ぐ方法]
```

---

## /analyze security

セキュリティリスクを監査し脆弱性を防止します。

### 1. 依存パッケージの脆弱性チェック
// turbo
```bash
npm audit 2>/dev/null || pip-audit 2>/dev/null
```

### 2. コードレビュー観点
- [ ] ハードコードされた認証情報がないか
- [ ] SQLインジェクション対策
- [ ] XSS対策
- [ ] CSRF対策
- [ ] 入力バリデーション
- [ ] 適切なエラーハンドリング

### 3. 設定ファイルの確認
- [ ] `.env` がgitignoreに含まれているか
- [ ] 機密情報が公開されていないか
- [ ] 本番環境の設定が適切か

### 4. 結果報告
```markdown
## 🔒 セキュリティ監査結果

### 脆弱性サマリー
| 深刻度 | 件数 |
|--------|------|
| Critical | X |
| High | X |
| Medium | X |
| Low | X |

### 発見された問題
1. [問題1]
   - 深刻度: [Critical/High/Medium/Low]
   - 対策: [対策内容]

### 推奨アクション
- [優先度高] [アクション1]
- [優先度中] [アクション2]
```
