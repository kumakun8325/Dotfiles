# 検証ワークフロー (Verify)

コード変更後は必ずこのワークフローを実行する。
「Claudeがフィードバックループを持っていれば、最終結果の品質は2〜3倍になる」

## 手順

### 1. ビルド確認
// turbo
プロジェクトのビルドを実行：
```bash
npm run build
# または
npm run tsc --noEmit  # TypeScriptの型チェックのみ
```

### 2. テスト実行
// turbo
テストスイートを実行：
```bash
npm test
# または
npm run test:unit
```

### 3. Lint確認
// turbo
コードスタイルの確認：
```bash
npm run lint
# または
npx eslint src/
```

### 4. 動作確認（該当する場合）

#### Webアプリの場合
- 開発サーバーを起動
- ブラウザで実際に操作して動作を確認
- 変更した機能が正しく動作することを確認

#### CLI/スクリプトの場合
- 変更したコマンドを実行
- 期待通りの出力が得られることを確認

### 5. 結果報告
検証結果をユーザーに報告：

```markdown
## ✅ 検証結果

| 項目 | 結果 | 詳細 |
|------|------|------|
| ビルド | ✅ / ❌ | [詳細] |
| テスト | ✅ / ❌ | [テスト数/失敗数] |
| Lint | ✅ / ❌ | [警告/エラー数] |
| 動作確認 | ✅ / ❌ | [確認内容] |
```

---

## 重要事項

- **検証せずにタスク完了としない**
- エラーがある場合は、報告前に修正を試みる
- 修正できない問題は明確にユーザーに報告する
