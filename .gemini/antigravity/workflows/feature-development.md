# 機能開発ワークフロー (Feature Development)

新機能の開発を計画から完了まで一貫してガイドする。

## 🚀 開発フロー概要

```
/plan → 開発 → /verify → /simplify → /commit
```

---

## 手順

### 1. ブランチ作成
// turbo
```bash
git checkout main
git pull origin main
git checkout -b feature/[機能名]
```

### 2. 計画フェーズ（/plan）
`/plan` ワークフローを実行：
- 目標の明確化
- 実装ステップの策定
- 影響範囲の分析
- 検証方法の決定

### 3. 実装フェーズ
計画に沿って段階的に実装：
- 1つのコミットで1つの論理的な変更
- 各ステップでテストを実行
- 問題があれば早期に報告

### 4. 検証フェーズ（/verify）
`/verify` ワークフローを実行：
- ビルド確認
- テスト実行
- Lint確認
- 動作確認

### 5. シンプル化フェーズ（/simplify）
`/simplify` ワークフローを実行：
- 不要なコードの削除
- 重複の排除
- コードの整理

### 6. コミット・PR作成（/commit）
`/commit` ワークフローを実行：
- 変更のコミット
- GitHubへのプッシュ
- PR作成（必要に応じて）

---

## 完了チェックリスト

### 機能要件
- [ ] すべての要件を満たしている
- [ ] エッジケースを考慮している
- [ ] エラーハンドリングが適切

### コード品質
- [ ] テストを追加した
- [ ] ドキュメントを更新した
- [ ] コードがシンプルで読みやすい
