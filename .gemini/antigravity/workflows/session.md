# 📁 セッション管理

セッション状態の保存と再開を管理します。

## 使い方

```
/session save   → セッションを保存して終了
/session resume → 前回の続きから再開
```

---

## /session save

作業終了時に以下のファイルを更新して状態を保存する。

### 1. セッション状態ファイルを更新
- ファイルパス: `勉強会_session_state.md`
- 更新内容:
  - Updated の日時を現在時刻に更新
  - Done に完了したタスクを追加
  - Now を現在の作業状態に更新
  - Next を次のアクションに更新

### 2. 関連ドキュメントを更新（該当する場合）
- テストサマリー、タスクリスト等を最新化
- 最終更新日を更新

### 3. Gitにコミット（任意）
- 変更をコミットする場合は `/git commit` を実行

### 4. 保存完了メッセージ
- 更新したファイル一覧を表示
- 次回再開時は `/session resume` を使用することを案内

---

## /session resume

前回のセッション状態を読み込んで作業を再開する。

### 1. 状態ファイルを読み込む
- ファイルパス: `勉強会_session_state.md`

### 2. 状態を表示
- 現在のプロジェクト名と目的
- 完了済みタスク（Done）
- 進行中タスク（Now）
- 次のアクション（Next）
- 関連ファイル

### 3. ユーザーに確認
- 「前回の続きから作業を再開します。何をしますか？」
